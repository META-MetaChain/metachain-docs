METAOS provides L2-specific precompiles with methods smart contracts can call the same way they can solidity functions. This reference page exhaustively documents the specific calls METAOS makes available through precompiles. For a more conceptual description of what precompiles are and how they work, please refer to the [precompiles conceptual page](/for-devs/concepts/precompiles.mdx).

This reference page is divided into two sections. The first one lists all precompiles in a summary table with links to the reference of the specific precompile, along with the address where they live, their purpose and links to the go implementation and solidity interface. The second one details the methods available in each precompile with links to the specific implementation.

## General information of precompiles

### METAAggregator

METAAggregator ([Interface][METAaggregator_link_interface] | [Implementation][METAaggregator_link_implementation]) provides aggregators and their users methods for configuring how they participate in L1 aggregation. metachain One's default aggregator is the Sequencer, which a user will prefer unless `SetPreferredAggregator` is invoked to change it.

Compression ratios are measured in basis points. Methods that are checkmarked are access-controlled and will revert if not called by the aggregator, its fee collector, or a chain owner.

### METABLS

:::caution Disabled

This precompile has been disabled. It previously provided a registry of BLS public keys for accounts.

:::

### METADebug

METADebug ([Interface][METAdebug_link_interface] | [Implementation][METAdebug_link_implementation]) provides mechanisms useful for testing. The methods of `METADebug` are only available for chains with the `AllowDebugPrecompiles` chain parameter set. Otherwise, calls to this precompile will revert.

import METADebugRef from './partials/precompile-tables/_METADebug.md';

<METADebugRef />

### METAFunctionTable

METAFunctionTable ([Interface][METAfunctiontable_link_interface] | [Implementation][METAfunctiontable_link_implementation]) provides aggregators the ability to manage function tables, to enable one form of transaction compression. The Nitro aggregator implementation does not use these, so these methods have been stubbed and their effects disabled. They are kept for backwards compatibility.

import METAFunctionTableRef from './partials/precompile-tables/_METAFunctionTable.md';

<METAFunctionTableRef />

### METAGasInfo

METAGasInfo ([Interface][METAgasinfo_link_interface] | [Implementation][METAgasinfo_link_implementation]) provides insight into the cost of using the chain. These methods have been adjusted to account for Nitro's heavy use of calldata compression. Of note to end-users, we no longer make a distinction between non-zero and zero-valued calldata bytes.

METAInfo ([Interface][METAinfo_link_interface] | [Implementation][METAinfo_link_implementation]) provides the ability to lookup basic info about accounts and contracts.

### METAosTest

METAosTest ([Interface][METAostest_link_interface] | [Implementation][METAostest_link_implementation]) provides a method of burning METAitrary amounts of gas, which exists for historical reasons. In Classic, `METAosTest` had additional methods only the zero address could call. These have been removed since users don't use them and calls to missing methods revert.

import METAosTestRef from './partials/precompile-tables/_METAosTest.md';

<METAosTestRef />

### METAOwner

METAOwner ([Interface][METAowner_link_interface] | [Implementation][METAowner_link_implementation]) provides owners with tools for managing the rollup. Calls by non-owners will always revert.

Most of metachain Classic's owner methods have been removed since they no longer make sense in Nitro:

- What were once chain parameters are now parts of METAOS's state, and those that remain are set at genesis.
- METAOS upgrades happen with the rest of the system rather than being independent
- Exemptions to address aliasing are no longer offered. Exemptions were intended to support backward compatibility for contracts deployed before aliasing was introduced, but no exemptions were ever requested.

### METARetryableTx

METARetryableTx ([Interface][METAretryabletx_link_interface] | [Implementation][METAretryabletx_link_implementation]) provides methods for managing retryables. The model has been adjusted for Nitro, most notably in terms of how retry transactions are scheduled. For more information on retryables, please see [the retryable documentation](METAos#Retryables).

METAStatistics ([Interface][METAstatistics_link_interface] | [Implementation][METAstatistics_link_implementation]) provides statistics about the chain as of just before the Nitro upgrade. In metachain Classic, this was how a user would get info such as the total number of accounts, but there are better ways to get that info in Nitro.
